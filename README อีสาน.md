# โปรอีสาน ซีรีย์ — Campaign Tracker 🍢

ฟีเจอร์ติดตามยอดขาย **3 เมนูอีสานซีรีย์** ข้าม **24 สาขา** สู่เป้ารวมบริษัท
**25,806 จาน** ระหว่าง **10 ก.ค. – 30 ก.ย. 2026** พร้อม **incentive ฿5/จาน**
คำนวณให้อัตโนมัติ

> ฟีเจอร์นี้ **รวมอยู่ในระบบ Santa Fe Sales Report 2.0 เดิมแล้ว** — ทำงานบน
> **Supabase + Google Apps Script** (ไม่ใช่แอปแยก/ไม่ใช้ Firebase) ทุกอย่างอยู่ใน
> ไฟล์ **`index.html`** ไฟล์เดียว

---

## 1) ภาพรวม — ข้อมูลไหลยังไง

1. **สาขา** กรอกจำนวนจานของ 3 เมนูอีสานตอน **ปิดการขายสิ้นวัน** (อยู่ในหน้า "ส่งยอดขาย"
   ส่วน 🎯 รายการเชียร์ขาย) — บันทึกลงตาราง `promo_submissions` (1 สาขา/1 วัน, แก้ไขได้)
2. **Google Apps Script** (`gas_sync_v20.gs` · `syncPromo` แบบ True Mirror) ซิงก์ข้อมูลจาก
   Google Sheet ↔ Supabase
3. **หน้า Dashboard** รวมยอดทุกสาขาแบบสด แสดง **การ์ดเป้าโปรอีสาน** + **รายการเชียร์ขาย**

ไม่มี **โควตารายสาขา** — 25,806 เป็น **เป้ารวมบริษัทเดียว** แต่ละสาขาแค่กรอกยอดจริงของตัวเอง

---

## 2) การ์ดเป้าโปรอีสาน (บน Dashboard)

การ์ด **🎯 เป้าโปรอีสาน ซีรีย์** แสดงเหนือ "รายการเชียร์ขาย" โดยใช้
**ยอดสะสมทั้งแคมเปญ ทุกสาขา** — *ไม่ขึ้นกับ date filter* ที่เลือกอยู่บน dashboard:

| ตัวเลข | ความหมาย |
|---|---|
| **Progress bar** | % ของเป้า 25,806 ที่ทำได้ (เขียว = ถึงเป้า) |
| **ยอดสะสม** | รวมจำนวนจานที่ทุกสาขาส่งมา ตั้งแต่ 10 ก.ค. ถึงวันนี้ |
| **เหลืออีก** | 25,806 − ยอดสะสม |
| **เหลือเวลา** | จำนวนวันจากวันนี้ถึง 30 ก.ย. 2026 (นับรวมวันนี้) |
| **ต้องขาย/วัน** | เหลืออีก ÷ เหลือเวลา (ปัดขึ้น) |
| **Badge ทันเป้า** | 🟢 ทันเป้า / 🟠 ต่ำกว่าเป้า — เทียบ %เป้าที่ทำได้ กับ %ช่วงเวลาที่ผ่านไป |
| **Incentive สะสม** | ยอดสะสม × ฿5 (ทั้งบริษัท) |

**สโคปการแสดงผล:** การ์ดโชว์ **เป้ารวมบริษัท** เสมอ และเมื่อเจาะดู **รายสาขา** จะเพิ่ม
บรรทัด "🏪 สาขานี้สะสม X จาน · incentive ฿Y" ให้ด้วย

## 4) ปรับค่าแคมเปญ (ที่เดียวจบ)

แก้ในค่า `PROMO_CAMPAIGN` ใกล้ต้นสคริปต์ของ `index.html`:

```js
const PROMO_CAMPAIGN = {
  goal: 25806,               // เป้าบริษัท (จาน)
  startDate: "2026-07-10",   // วันเริ่มโปร
  endDate:   "2026-09-30",   // วันสิ้นสุดโปร (นับรวมวันนี้)
  incentivePerDish: 5        // incentive ฿ ต่อจาน
};
```

- **เปลี่ยนเป้า / วันที่ / incentive** — แก้ 4 ค่านี้
- **เปลี่ยน/เพิ่มเมนู** — แก้ `PROMO_ITEMS`
- คำนวณ days-left / ต้องขายต่อวัน / on-pace / incentive ใหม่ให้อัตโนมัติ

## 5) การตั้งค่าฝั่งแอดมิน

เก็บในตาราง `app_settings` (คีย์/ค่า) อ่านผ่าน `_loadPromoEnabled()`:

- **`promo_enabled`** = `true` / `false` — เปิด/ปิดการเชียร์ขายทั้งระบบ
  (ปิด = ไม่มีสาขาไหนเห็นการ์ดเชียร์ขาย และการ์ดเป้าก็ซ่อน)
- **`promo_branches`** = `"ALL"` หรือ JSON array ของ branch code เช่น `["5001","5023"]`
  — ระบุว่าสาขาไหนเห็นฟอร์มเชียร์ขาย

ปรับได้จากหน้า "ตั้งค่ารายการเชียร์ขาย" ในแอป

## 6) โครงสร้างข้อมูล (Supabase)

- **`promo_submissions`** — 1 แถวต่อสาขาต่อวัน: `branch_code`, `submit_date`, `items` (JSONB:
  `[{id,name,price,qty,subtotal}]`), `total_qty`, `total_amount` · UNIQUE(branch_code, submit_date)
- แก้ไขได้ (มี `updated_at`, `edit_count`, `last_edited_by` — trigger `promo_bump_edit`)
- SQL อ้างอิง: [promo_schema.sql](promo_schema.sql), [promo_v2_schema.sql](promo_v2_schema.sql),
  [promo_v3_schema.sql](promo_v3_schema.sql)

---

## 7) จุดสำคัญในโค้ด (`index.html`)

| ส่วน | หน้าที่ |
|---|---|
| `PROMO_ITEMS` / `PROMO_CAMPAIGN` | ค่าคอนฟิกเมนู + แคมเปญ |
| `_loadPromoCampaign()` | ดึงยอดสะสมทั้งแคมเปญ (company-wide) จาก `promo_submissions` |
| `_renderPromoCampaign()` | วาดการ์ดเป้า + incentive |
| `_renderPromoStrip()` | วาด "รายการเชียร์ขาย" (ตามช่วงที่เลือก) |
| `_readPromoItems()` | อ่านค่าที่สาขากรอกในฟอร์มสิ้นวัน |

---

## 8) หมายเหตุ

- **เหลือเวลา** นับถอยหลังถึง 30 ก.ย. 2026 — เปิดหลังวันนั้นจะขึ้น 0 (ปกติ)
- ยอด/incentive คิดจากข้อมูลจริงในฐานข้อมูล — แก้ยอดย้อนหลังของสาขาได้ ตัวเลขอัปเดตตาม
- การ์ดเป้าจะซ่อนถ้า `promo_enabled = false`
